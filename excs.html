num_cols = list(data.select_dtypes(include=np.number).columns)

def plot_chunk(cols):
    """
    Plota at 3 histogramas para a lista de colunas `cols`
    """
    n = len(cols)
    fig, axes = plt.subplots(1, n, figsize=(5*n, 5))
    # se for s 1 coluna, garantir itervel
    if n == 1:
        axes = [axes]
    for ax, col in zip(axes, cols):
        sns.histplot(data[col], kde=True, bins=20, ax=ax)
        colName = col.capitalize().replace('_', ' ')
        ax.set_title(f"Histogram Attribute  {colName}")
        ax.set_xlabel(f"{colName} level")
        ax.set_ylabel("Frequency")
        ax.grid(True)
    fig.subplots_adjust(wspace=0.4)
    plt.show()

Escolhi a coluna RestingECG, que registra o tipo de eletrocardiograma em repouso com trs categorias possveis: Normal, ST e LVH. Primeiro, apliquei o Label Encoding, que simplesmente atribui a cada categoria um nmero inteiro sem nenhuma relao de ordem real entre eles. No meu caso, o scikit-learn mapeou LVH 0, Normal 1 e ST 2, gerando uma nica coluna nova chamada, por exemplo, ECG_label.
Em seguida, usei o One-Hot Encoding, que cria uma coluna binria para cada categoria, indicando com 1 onde a observao pertence quela categoria e com 0 nos demais casos. Usando o pd.get_dummies, obtive trs colunas: ECG_LVH, ECG_Normal e ECG_ST.
Resumindo, o Label Encoding gera 1 coluna com valores inteiros (0, 1, 2), enquanto o One-Hot gera 3 colunas binrias (uma para cada categoria). Para algoritmos baseados em distncia (como k-NN ou k-means), o One-Hot Encoding costuma ser mais adequado: ele trata cada categoria como um eixo independente, garantindo que a distncia entre quaisquer duas categorias seja sempre a mesma e no seja influenciada por um suposto ranking que o Label Encoding impe (por exemplo, fazer com que ST (2) parea duas vezes mais distante de LVH (0) do que Normal (1)).

age = data['Age']

age_equal = pd.cut(age, bins=5)

age_quantile = pd.qcut(age, 5)

fig, axes = plt.subplots(1, 3, figsize=(18, 4))

# histograma original
axes[0].hist(age, bins=15, edgecolor='black')
axes[0].set_title('Age  Distribuio Original')
axes[0].set_xlabel('Idade')
axes[0].set_ylabel('Frequncia')

# histograma com intervalos uniformes
counts_eq = age_equal.value_counts().sort_index()
axes[1].bar([str(iv) for iv in counts_eq.index], counts_eq.values, edgecolor='black')
axes[1].set_title('Age  Bins de Largura Igual')
axes[1].tick_params(axis='x', rotation=45)

# histograma com intervalos por quantis
counts_qt = age_quantile.value_counts().sort_index()
axes[2].bar([str(iv) for iv in counts_qt.index], counts_qt.values, edgecolor='black')
axes[2].set_title('Age  Bins por Quantis')
axes[2].tick_params(axis='x', rotation=45)
plt.tight_layout()
plt.show()

A tcnica de quantis gera grupos mais equilibrados em termos de nmero de observaes, o que til quando queremos comparar subamostras de tamanho similar (por exemplo, testar modelo ou analisar comportamentos por faixa com equal-freq). J a discretizao uniforme ajuda a ver claramente como os valores se distribuem ao longo do eixo de idades, destacando faixas esparsas ou concentradas ideal para explorar vacncias ou picos na escala original.
